# Guía AGENTS

Este archivo es intencionalmente minimalista.

Toda la guía operativa para agentes de código está documentada en `docs/`.

Empieza aquí:

- `docs/README.md`

Mapa principal de documentación:

- `docs/00-overview/project-overview.md`: visión general del proyecto, objetivos, alcance y contexto funcional.
- `docs/01-setup/local-setup.md`: pasos de instalación local, dependencias y arranque del entorno.
- `docs/02-architecture/source-layout.md`: estructura del código fuente, carpetas clave y convenciones de organización.
- `docs/03-features/agent-guidelines.md`: lineamientos de trabajo para agentes, flujo recomendado y buenas prácticas.
- `docs/04-components/ui-guidelines.md`: guía de componentes UI, patrones visuales y criterios de consistencia.
- `docs/05-testing/testing-commands.md`: comandos de testing, tipos de pruebas y ejecución local/CI.
- `docs/06-deploy/quality-gates.md`: validaciones previas a despliegue y criterios de calidad obligatorios.
- `docs/07-decisions/coding-standards.md`: estándares de código, decisiones técnicas y reglas de estilo.
- `docs/08-runbooks/agent-runbook.md`: runbook operativo del agente para tareas comunes e incidencias.
